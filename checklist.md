# Checklist — SauceDemo

## Project Information

- **Project:** SauceDemo
- **Date:** 20.03.2026
- **Build:** 1.0
- **Tester:** Kirill Usmanov
- **Environment:** macOS Sequoia 15.2, Google Chrome 146.0.7680.81

---

## Авторизация

- Наличие полей Username, Password и кнопки Login
- Возможность авторизоваться с валидными данными (standard_user / secret_sauce)
- Невозможность авторизоваться с невалидными данными (несуществующий логин, неверный пароль)
- Сообщение об ошибке при вводе верного имени пользователя и неверного пароля
- Сообщение об ошибке при вводе неверного имени пользователя
- Сообщение об ошибке при попытке авторизоваться заблокированным пользователем (locked_out_user / secret_sauce)
- Перенаправление на страницу каталога товаров после успешной авторизации
- При нажатии на кнопку Logout пользователь выходит из системы и возвращается на страницу авторизации

---

## Каталог товаров

- Возможность корректной сортировки товаров по цене (low to high, high to low)
- Возможность корректной сортировки товаров по имени (A to Z, Z to A)
- Сохранение параметров сортировки при обновлении страницы
- При нажатии на кнопку Add to cart происходит добавление товара в корзину, кнопка меняется на Remove
- При нажатии на кнопку Remove происходит удаление товара из корзины, кнопка меняется на Add to cart
- Возможность добавить несколько товаров в корзину
- Счетчик товаров в корзине отображает количество товаров в корзине
- При нажатии на иконку корзины происходит переход в корзину

---

## Корзина

- Информация о товарах в корзине соответствует добавленным товарам из каталога
- При нажатии на кнопку Remove товар удаляется из корзины, счетчик обновляется соответствующим образом
- При нажатии на кнопку Continue Shopping происходит переход на страницу каталога товаров
- После выхода из системы доступ к странице корзины невозможен без повторной авторизации
- При обновлении страницы корзины товары сохраняются

---

## Оформление заказа

- Возможность оформления заказа с валидными данными
- Невозможность оформления заказа с невалидными данными
- Возникновение ошибки при вводе невалидных данных (цифры/спецсимволы в имени и фамилии, буквы в поле zip-кода)
- Возникновение ошибки при пустых обязательных полях
- При нажатии на кнопку Cancel происходит переход на страницу корзины
- При вводе валидных данных и нажатии на кнопку Continue происходит переход на страницу подтверждения заказа

---

## Подтверждение оформления заказа

- Товары в подтверждении соответствуют фактически оформленным товарам
- Сумма заказа соответствует сумме оформленных товаров
- При нажатии на кнопку Cancel происходит переход на страницу оформления заказа
- При нажатии на кнопку Finish происходит переход на страницу с уведомлением об успешном оформлении заказа
